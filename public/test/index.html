test

declare the tools that must exist and how their calls must behave. mctop test connects, checks the contract, and exits non-zero when it drifts, so a renamed tool fails the build instead of an agent in production.

example

# spec.yaml
expect:
  tools: [search, fetch]
calls:
  - tool: search
    assert: { not_error: true, contains: "result" }

$ mctop test spec.yaml
✓ tools present   ✓ search not_error   ✓ search contains
3 passed, 0 failed   exit 0

specs are strict yaml: unknown keys are errors, so a typo never silently passes. assert on not_error, substring contains, and the tools a server must expose. add --report json for machine-readable ci output.
